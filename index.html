Practical GitHub for Version Control
A Hands-on Session

Git is a distributed version control system created by Linus Torvalds in 2005 for managing projects of all sizes with speed and efficiency. It tracks changes, maintains a comprehensive project history, and allows multiple developers to collaborate seamlessly. One of its key features is branching, enabling independent lines of development that can be merged back into the main project. Git also supports lightweight tags, marking important points like version releases. As a distributed system, every developer has a complete copy of the repository, ensuring work continuity even without a central server.

GitHub, a web-based platform built on Git, enhances these capabilities with a user-friendly interface and powerful collaboration tools, making it the largest host of source code globally. It features pull requests for proposing changes, code reviews, and merging, alongside robust issue tracking for managing project workflows.

Getting Started with Git and GitHub

Install Git: You can download Git from git-scm.com.

Basic Git Commands:

  • git init - Initialize a new Git repository.
  • git clone [URL] - Clone an existing repository.
  • git add [file] - Stage a file for the next commit.
  • git commit -m "message" - Commit staged files with a descriptive message.
  • git status - Check the status of your working directory.
  • git push origin [branch-name] - Push your changes to the remote repository.
  • git pull - Fetch and merge changes from the remote repository.
